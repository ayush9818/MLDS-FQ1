You will analyze data from a website with 8 pages (plus a 9th state, indicating that the user has left the website). Each visitor starts at the home page (Page 1), then browses from page-to-page until he/she leaves the website. So, a sample path may be S1 S3 S5 S9, corresponding to a visitor starting on the home page, moving to Page 3, then Page 5, then leaving the website.

Attached is the dataset webtraffic.txt, which records the paths of 1000 visitors (rows). The data has 81 columns labeled t11, t12, , t19, t21, t22, , t99, where tij represents a transition from State i to State j, for i, j {1, , 9}. Each visitor has a 1 in column tij if the visitor clicked from Page i to Page j, and 0 elsewhere. For example, the aforementioned sample path would have 1s in columns t13, t35, and t59 and 0s elsewhere.

1b)¶

Draw a directed graph where each node represents a state, and each arrow from State i to State j has positive (non-zero) traffic (i.e., Traffic[i,j]>0). This may be submitted as a TikZ graph (or using your graphing program of choice) or a picture of a hand-drawn graph (provided it is legible). Is the Markov chain irreducible? Is the Markov chain ergodic? Explain.

Ans:

  • Irreducibility: A Markov chain is irreducible if all states communicate with each other in a finite number of steps. Looking at the provided graph: every node seems to be connected either directly or through a series of other nodes. Thus, based on the graph, the Markov chain appears to be irreducible.

  • Ergodic: A state is ergodic if it is positive recurrent and aperiodic. A Markov chain is ergodic if all its states are ergodic and it is irreducible. We have established that this markov chain is irreducible. Since states 1 & 9 are not aperiodic, the entire Markov Chain is not aperiodic and thus not Ergodic.

state_after_n_steps <- function(trans_prob, start_state, target_state, num_steps) {
    trans_prob_powered = trans_prob %^% num_steps  # Raising P to the power of "clicks"
    return(trans_prob_powered[start_state, target_state])

steady_state <- function(P) {
  n <- nrow(P)
  # Construct the system of equations
  Q <- t(P - diag(n))
  
  # Add the constraint that probabilities sum to 1
  Q <- rbind(Q, rep(1,n))
  
  b <- rep(0, n)
  b[n + 1] <- 1
  
  # Solve the system of equations
  pi <- qr.solve(Q,b)
  return(pi)
}

mean_first_passage_time <- function(P, time_matrix){
    n <- nrow(P)
    
    # Modify the transition matrix to make it an n-1xn-1 matrix
    # (if the transition to the terminal state is not already excluded)
    P_modified = P[1:n-1, 1:n-1]
        
    # Identity matrix of size n-1 x n-1
    I = diag(n-1)
    A = I - P_modified
    b = time_matrix[1:n-1]
    
    t = solve(A, b)
    average_time = t[1]
    return(average_time)
}

Problem 2a¶

Determine the number of samples required to achieve an error tolerance of 1e-3 with 99% confidence.

Number of samples needed to ( n ) to meet tolerance with ( (1 - \delta) ) confidence:
$$ n \geq \frac{\text{var}[p(x)]}{\text{tolerance}^2 \cdot \delta} $$

Given the values:

  • $\text{var}[p(x)] = \frac{1}{{\lambda}^2}$
  • $\text{tolerance} = 10^{-3}$
  • $\delta = 0.01$

Substituting into the formula and simplifying, we get: $$ n \geq 10^{8} \times \frac{1}{{\lambda}^2} $$

Problem 3b¶

Using a burn-in period of 5000 samples and keeping every 100 samples, generate 100 samples from the gamma distribution with shape k = 2 and scale = 2. Use the algorithm you chose in Problem 3a and write your own sampler (as opposed to using a function from a package).

gamma_pdf = function (x,k,theta){
  return((x^(k - 1) / (gamma(k) * theta^k)) * exp(-x / theta))
}

metropolis_hastings_simple <- function(n_samples, burn_in, thin, lambda_init, k, theta) {
  # Initialize
    
  samples <- numeric(burn_in + n_samples * thin)
    
  # Initialising the initail sample with rate lambda init is a design choice here
  samples[1] <- rexp(1, rate = lambda_init)

  for (i in 2:(burn_in + n_samples * thin)) {
    # For each iteration, we propose a new sample from an exponential distribution with a rate equal
    # to the last accepted sample. This ensures the chain can explore different regions of the target distribution.
    proposed_sample <- rexp(1, rate = samples[i-1])
    
    # Compute the acceptance ratio
    alpha = gamma_pdf(proposed_sample,k,theta) / gamma_pdf(samples[i-1],k,theta)
    
    # If a uniformly drawn random number is less than alpha, we accept it; otherwise, we keep the current sample.
    if (runif(1) < min(1,alpha)) {
      samples[i] <- proposed_sample
    } else {
      samples[i] <- samples[i-1]
    }
  }

  # After generating all the samples, we discard the first burn_in samples. 
  # Then, we apply thinning to reduce autocorrelation by only keeping every nth sample,
  # where n is specified by thin.
  return(samples[(burn_in + 1):length(samples)][seq(1, n_samples * thin, by = thin)])
}

# Generate samples
